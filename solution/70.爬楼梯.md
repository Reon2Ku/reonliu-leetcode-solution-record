# 70.爬楼梯

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

 

**示例 1：**

```
输入：n = 2
输出：2
解释：有两种方法可以爬到楼顶。
1. 1 阶 + 1 阶
2. 2 阶
```

**示例 2：**

```
输入：n = 3
输出：3
解释：有三种方法可以爬到楼顶。
1. 1 阶 + 1 阶 + 1 阶
2. 1 阶 + 2 阶
3. 2 阶 + 1 阶
```



## 题解

对于动态规划问题我们首先要找规律。所谓动态规划，指的是下一个变量的结果可以由前面已确定的一个，或两个，或多个结果推导而来。也许你一眼看不出推导关系，但当你把一个个结果经过计算后列出来，自然会发现其中规律。本体仅看题目描述还不好思考，但当你从1开始计算有几种方法可以爬到楼顶时，你会发现这就是一个斐波那契数列。

~~~
class Solution {
public:
    int climbStairs(int n) {
        vector<int> dp(n+1,0);
        //cout<<dp.size();
        if(n<=3) return n;
        dp[1] = 1;
        dp[2] = 2;
        for(int i=3;i<n+1;i++)
        {
            dp[i] = dp[i-1] + dp[i-2];
        }
        return dp[n];
    }
};
~~~

如果说还有优化空间的话，那就是斐波那契数列仅需记录两个数字的状态即可，不需要记录整个数组。但那样就没法提现动态规划的思维了
