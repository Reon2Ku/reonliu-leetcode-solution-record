# 62.不同路径

一个机器人位于一个 `m x n` 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？

 

**示例 1：**

![img](https://pic.leetcode.cn/1697422740-adxmsI-image.png)

```
输入：m = 3, n = 7
输出：28
```

**示例 2：**

```
输入：m = 3, n = 2
输出：3
解释：
从左上角开始，总共有 3 条路径可以到达右下角。
1. 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右
3. 向下 -> 向右 -> 向下
```

**示例 3：**

```
输入：m = 7, n = 3
输出：28
```

**示例 4：**

```
输入：m = 3, n = 3
输出：6
```



## 题解

呃呃这道题其实转化一下就是求在m+n-2个数中选择m-1个数，也就是C(m+n-2)(m-1), 但直接用数学方法做的话似乎又不太行，用int会超过int最大值，用double会导致乘除之后结果不准确。

~~~
class Solution {
public:
    int uniquePaths(int m, int n) {
        long result = 1;
        long count = m-1;
        while(count>0)
        {
            result*=m+n-2-(m-1-count);
            count--;
        }
        count = m-1;
        while(count>0)
        {
            result/=count;
            count--;
        }
        return int(result);
    }
};
~~~



牛魔的long long都超出最大限制，给我逼急眼了，我直接想出标准答案

~~~
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> result(m,vector<int>(n,1));
        for(int i=1;i<m;i++)
        {
            for(int j=1;j<n;j++)
            {
                result[i][j] = result[i-1][j]+result[i][j-1];
            }
        }

        return result[m-1][n-1];
    }
};
~~~
