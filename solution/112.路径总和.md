# 112.路径总和

## 题目描述

给你二叉树的根节点 `root` 和一个表示目标和的整数 `targetSum` 。判断该树中是否存在 **根节点到叶子节点** 的路径，这条路径上所有节点值相加等于目标和 `targetSum` 。如果存在，返回 `true` ；否则，返回 `false` 。

**叶子节点** 是指没有子节点的节点。

 

**示例 1：**

![img](https://assets.leetcode.com/uploads/2021/01/18/pathsum1.jpg)

```
输入：root = [5,4,8,11,null,13,4,7,2,null,null,null,1], targetSum = 22
输出：true
解释：等于目标和的根节点到叶节点路径如上图所示。
```

**示例 2：**

![img](https://assets.leetcode.com/uploads/2021/01/18/pathsum2.jpg)

```
输入：root = [1,2,3], targetSum = 5
输出：false
解释：树中存在两条根节点到叶子节点的路径：
(1 --> 2): 和为 3
(1 --> 3): 和为 4
不存在 sum = 5 的根节点到叶子节点的路径。
```

**示例 3：**

```
输入：root = [], targetSum = 0
输出：false
解释：由于树是空的，所以不存在根节点到叶子节点的路径。
```



## 题解

参考了所有路径写的，双stack

~~~
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    bool hasPathSum(TreeNode* root, int targetSum) {
        stack<TreeNode*> sta;
        stack<int> path;
        if(root==NULL) return false;
        sta.push(root);
        path.push(root->val);
        int result;
        while(!sta.empty())
        {
            TreeNode* node = sta.top();
            sta.pop();
            result = path.top();
            path.pop();

            if(!node->left&&!node->right)
            {
                if(result==targetSum) return true;
                else continue;
            }
            if(node->left) 
            {
                sta.push(node->left);
                path.push(result+node->left->val);
            }
            if(node->right)
            {
                sta.push(node->right);
                path.push(result+node->right->val);
            }
        }
        return false;
    }
};
~~~

接下来凭自己记忆写的递归：

~~~
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    void myPath(TreeNode* node,int result,bool& flag,int target)
    {
        if(!node->left&&!node->right)
        {
            if(result==target) flag = true;
            return;
        }
        if(node->left)
        {
            myPath(node->left,result+node->left->val,flag,target);
        }
        if(node->right)
        {
            myPath(node->right,result+node->right->val,flag,target);
        }
    }
    bool hasPathSum(TreeNode* root, int targetSum) {
        if(root==NULL) return false;
        bool flag = false;
        int result = root->val;
        myPath(root,result,flag,targetSum);
        return flag;
    }
};
~~~

我靠，怎么还有这么巧妙的办法。我说怎么我明明按照正常的思路去分析去写代码写了一个O(n)却还是会时耗超高，估计是在声明各种变量、进行各种对比的时候花费了不少时间。

我们来看超级极简的递归代码，4ms击败90%。写出这么巧妙代码的也是神人了。

不过对于做题来说，重要的是分析清楚里面的过程，掌握思路，首先做出来之后再去简化代码。

~~~
class Solution {
public:
    bool hasPathSum(TreeNode* root, int sum) {
        if (!root) return false;
        if (!root->left && !root->right && sum == root->val) {
            return true;
        }
        return hasPathSum(root->left, sum - root->val) || hasPathSum(root->right, sum - root->val);
    }
};
~~~
